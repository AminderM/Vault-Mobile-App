## Task 2: Credentials Setup

### 2A: EXPO_TOKEN (For GitHub Actions CI/CD)

**What it is:** Token that allows GitHub Actions to trigger EAS builds

**Steps:**
1. Go to https://expo.dev/login
2. Sign in with your Expo account
3. Go to https://expo.dev/settings/tokens
4. Click "Create Token"
5. Name it: `github-actions-token`
6. Copy the token

**Store in GitHub:**
1. Go to your repo: https://github.com/AminderM/Vault-Mobile-App
2. Settings → Secrets and variables → Actions
3. Click "New repository secret"
4. Name: `EXPO_TOKEN`
5. Paste the token value
6. Click "Add secret"

**Verify:**
```bash
# This will work once EXPO_TOKEN is set in GitHub
gh secrets list -R AminderM/Vault-Mobile-App
```

### 2B: Apple ID (For iOS Submission)

**Prerequisites:**
- Apple Developer Account ($99/year)
- Apple ID with admin access

**What we need:**
1. **Apple ID:** aminderpro@gmail.com
2. **Password:** (store securely in GitHub Secrets)
3. **Team ID:** (from Apple Developer)
4. **Bundle ID:** com.integraatech.vault

### 2C: Google Play (For Android Submission)

**Prerequisites:**
- Google Play Developer Account ($25 one-time)
- Google Cloud Project

**What we need:**
1. **Package Name:** com.integraatech.vault
2. **Service Account JSON:** (for automated publishing)
3. **Signing Key:** (generated by EAS)

**Get Service Account JSON:**
1. Go to https://console.cloud.google.com
2. Create a new project or select existing
3. APIs & Services → Service Accounts
4. Create a new service account
5. Grant role: "Editor"
6. Create JSON key
7. Download the JSON file

**Store in GitHub Secrets:**
```
GOOGLE_PLAY_KEYSTORE_JSON: (paste entire JSON content)
GOOGLE_PLAY_PACKAGE: com.integraatech.vault
```

**Or use Base64:**
```bash
# Encode the JSON file
base64 -i path/to/keystore.json

# Store as GOOGLE_PLAY_KEYSTORE_BASE64 in GitHub
```

## Task 3: Production Builds

### Build for Android

**Step 1: Update EAS Configuration**

Verify in `eas.json`:
```json
{
  "build": {
    "production": {
      "distribution": "store",
      "android": {
        "buildType": "app-bundle",
        "gradleCommand": ":app:bundleRelease"
      }
    }
  }
}
```

**Step 2: Build**
```bash
# Build for Android (creates AAB for Play Store)
eas build --platform android --profile production

# This will:
# 1. Build in the cloud
# 2. Sign with your credentials
# 3. Create .aab file (Android App Bundle)
# 4. Output link to download
```

**Expected Output:**
```
✅ Android app bundle built successfully
📦 Download: https://expo-builds.s3.amazonaws.com/...aab
```

### Build for iOS

**Step 1: Verify iOS Configuration**

In `eas.json`:
```json
{
  "build": {
    "production": {
      "distribution": "store",
      "ios": {
        "buildConfiguration": "Release"
      }
    }
  }
}
```

**Step 2: Build**
```bash
# Build for iOS (creates IPA for App Store)
eas build --platform ios --profile production

# This will:
# 1. Build in the cloud on macOS
# 2. Sign with Apple certificates
# 3. Create .ipa file (iOS app archive)
# 4. Output link to download
```

**Expected Output:**
```
✅ iOS app archive built successfully
📦 Download: https://expo-builds.s3.amazonaws.com/...ipa
```

### Build Both Platforms

```bash
# Build both at once (runs in parallel)
eas build --platform all --profile production

# Monitor progress:
# 1. Go to https://expo.dev/builds
# 2. Watch both builds complete
# 3. Download when ready
```

**Estimated Time:**
- Android: 10-15 minutes
- iOS: 20-30 minutes (slower due to macOS environment)

## Troubleshooting

### Build Fails with Credentials Error

```bash
# Verify EXPO_TOKEN
echo $EXPO_TOKEN

# If empty, export it locally:
export EXPO_TOKEN="your-token-here"

# Try build again
eas build --platform all --profile production
```

### App Store Rejects for Privacy Policy

**Solution:**
1. Add privacy policy URL to your website
2. Update app.json:
```json
{
  "plugins": [
    ["expo-build-properties", {
      "ios": {
        "infoPlist": {
          "NSPrivacyTracking": false
        }
      }
    }]
  ]
}
```

### Google Play Rejects for Target API

**Solution:**
1. Update targetSdkVersion in app.json
2. Rebuild with `eas build --platform android --profile production`

### Review Takes Too Long

- Google Play: Usually 2-4 hours, sometimes 24h for first-time apps
- Apple App Store: Usually 24-48 hours

Normal for first submission. Subsequent updates are faster.
